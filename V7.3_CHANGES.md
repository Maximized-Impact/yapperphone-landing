Be first to know." is visible. Honoured by both smooth scroll and the `prefers-reduced-motion: reduce` snap path.

## 1. Live Founders counter (Firestore read)
- `src/firebase.js`: added `getFirestore`, `doc`, `getDoc` imports + `getFoundersCount()` helper. Reads `counters/originals.count` (public per `firestore.rules:103`). Returns the integer count of CLAIMED founders, or `null` on any failure.
- `src/App.jsx`: single `useEffect` on `LandingPage` mount fires one Firestore read; result stored in `foundersCount` state; `nextFounderNumber = foundersCount + 1` passed down to three render slots.
- Three placements (no per-component fetches):
  - **Hero CTA second line** (pre-launch only) — appended below "Founders get access first" as a separate green `micro-text` line.
  - **Sticky top banner** — preserves "🟢 YAPPER FOUNDERS" brand prefix; folds count into the body: `be #N of the first 1,000 · Lifetime €67`.
  - **Founders card** (§9) — adds "Be Founder #N of 1,000" line directly under the €67 price.
- Fallback: if the read fails or returns no number, all three slots silently revert to the static "First 1,000" copy. Never shows 0 or a wrong number.
- Off-by-one: counter holds CLAIMED count (=5 today, from seeded founders #1–5). Display = `count + 1` = `#6` = the next founder.
- Gift modal Founders tier tile intentionally NOT touched — gifter isn't becoming a founder; `/gift` already renders the landing's hero/banner/card via `<LandingPage giftAutoOpen />`.

## 2. Confirm recipient email + match validation (GiftModal)
- Added `recipientEmailConfirm` to form state.
- New "Confirm recipient's email" field appears below the existing recipient-email row, full-width.
- Match comparison uses `.trim()` on both sides — defeats autofill trailing-whitespace silent-block. Case-sensitive (intentional: confirm fields catch typos).
- Mismatch state: red border + "Emails don't match." message + `aria-invalid="true"`.
- Send button disabled until `emailsMatch === true`.
- `handleSubmit` also guards on `emailsMatch` (belt-and-suspenders).
- `onPaste` prevented on the confirm field — forces re-typing, which actually catches mistypes.

## 3. Monthly tier auto-select on month-button click (GiftModal)
- One-line fix: `onClick={(e) => { e.stopPropagation(); setTier('monthly'); setMonths(m) }}` — `setTier('monthly')` added before `setMonths(m)`.
- `stopPropagation` retained so clicking the inner button doesn't bubble to the tile's own `onClick`.
- Previously: default tier = `'annual'`, clicking "1mo" set months but tier stayed annual → Send button read €19.99 even though the visible month-button was active. Now both stay in sync.

## 4. Responsive hamburger drawer (≤580px)
- New `MobileNavDrawer` component: right-to-left slide-in `<aside id="mobile-nav-drawer">`, `min(280px, 86vw)` wide, full-height (`100dvh`), dark glassy panel with `backdrop-filter: blur(14px)`.
- Hamburger button placed on the navbar right (after the existing CTA, but the CTA hides at ≤580px so the button effectively replaces it).
- `aria-label="Open menu"`, `aria-expanded={mobileNavOpen}`, `aria-controls="mobile-nav-drawer"` — the controls reference resolves to the aside's matching `id`.
- ESC closes (keydown listener inside drawer's open effect); backdrop tap closes; link tap closes via `onClick={onClose}`.
- Plain `<a href>` anchors: `/features`, `/gift`, `/privacy` — matches existing nav pattern; the path router handles them on full-page load.
- Pre-launch CTA = `#founders` (matches desktop navbar `App.jsx:336`). Flips to Play Store when `LAUNCHED` flips true.
- Local `useState` only; no localStorage. Body-scroll-lock intentionally skipped (drawer + backdrop fully obscure page).
- CSS scoped under existing `@media (max-width: 580px)` breakpoint; respects `prefers-reduced-motion: reduce`.
