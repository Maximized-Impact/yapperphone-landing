# Yapper site — V7.1 changes

V7.1 takes the site out of the false "launched" state (the old date-driven gate had
passed, so the live site was showing dead Play/Samsung CTAs) and re-frames everything
for the in-review window: Founders €67 as the primary action, Discord + email waitlist
as the free path. Also enforces canonical naming/claims.

## The launch switch
`const LAUNCHED = false` now lives at the top of `App.jsx`, `FeaturesPage.jsx`, `LegalPage.jsx`.
Flip all three to `true` the moment Google approves → restores Play + 7-day-trial CTAs everywhere.
(The old countdown/`LAUNCH_DATE` machinery was removed; no fixed date is shown.)

- **CTAs (in-review):** hero/final/sticky/nav now lead with **Become a Founder — €67** (→ Stripe)

  `joinWaitlist` callable in `firebase.js`). Both Discord + email, as requested.

  **Gift Founders**; "Lifetime Pro" → "Lifetime access". *Backend tier key stays `originals`
  so `createGiftCheckout` / `gift-success.html` keep working unchanged.*

- **Canonical naming:** Yapper Originals → **Yapper Founders** (incl. legal/terms body);
  **Lifetime Pro → Lifetime access**; **"SUPER SINCE BIRTH Tmi" → "SUPER SINCE BIRTH"**
  (incl. legal body; entity-type is still described as "sole proprietorship (toiminimi)").
  T1D pledge: "free lifetime Pro" → "free lifetime access to Yapper Phone".
